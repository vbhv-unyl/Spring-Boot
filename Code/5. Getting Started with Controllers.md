Now the Question: How Did This Work?
When you typed a URL:

Your browser sent a request
Spring Boot received that request
A specific class handled it
A method returned a response
That class is called a Controller.

What Is a Controller in Spring Boot?
A Controller is a class in Spring Boot that:

Listens for incoming HTTP requests
Decides what code should run
Sends a response back to the client
In simple words:

Controller = Traffic controller for URLs
Whenever a request comes in, the controller tells Spring:
“For this URL, run this method.”
